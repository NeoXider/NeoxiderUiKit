# Конвенции имён Figma — контракт дизайнера

Сканер kit'а (`FuiUxmlScanner`) строится **на классах `fui_type_*`**, которые проставляет импортёр «MTK Figma UI Import»; `name` элемента — только идентификатор. Чем аккуратнее дизайнер именует слои в Figma, тем больше kit собирает автоматически. Этот файл — контракт: правила + что деградирует при их нарушении.

## Экраны

| Правило | Пример |
|---|---|
| Корень экрана: класс `fui_type_screen` (+ `fui_screen_<name>`), name = id страницы | `loading`, `mainmenu`, `gameplay` |

- pageId берётся из класса `fui_screen_<name>`, иначе из `name` корня.
- Имя страницы попадает в C#-класс: `mainmenu` → `MainmenuView`, константы `UiIds.Mainmenu.*`.

**Без правила:** файл без `fui_type_screen` пропускается сканером с предупреждением — страница не существует для kit'а.

## Попапы

| Правило | Пример |
|---|---|
| Корень попапа: класс `fui_type_popup`, name `popup_<x>` | `popup_setting`, `popup_pause`, `popup_endgame` |
| Внутри — полноэкранная подложка с name `background` | становится dim-затемнением (`uikit-dim`) |
| Контент попапа — `panel_<x>`, заголовок — `panel_header` | `panel_header` получает intro-анимацию |

- Попап скрывается kit'ом до первого кадра (в импортированном UXML он видим и его `background` перехватывал бы ввод).
- Безымянный `fui_type_popup` пропускается с предупреждением.
- Кнопки внутри попапа при `OpenAsync()` возвращают результат: имя кнопки без префикса `button_` (`button_restart` → `"restart"`), переопределяется в конфиге.
- Контентные `panel_*` внутри попапа — НЕ попапы; инлайн-карточки (как `panel_upgrade` в меню) тоже.

**Без правила:** элемент без `fui_type_popup` не станет `PopupView` — ни скрытия при старте, ни Open/Close/OpenAsync, ни dim. Кнопка `popup_*`-имени недостаточно — нужен именно класс.

## Кнопки

| Правило | Пример |
|---|---|
| Кнопка: класс `fui_type_button` (импортёр делает элемент `Button`), name `button_<действие>` | `button_play`, `button_close`, `button_buy` |

Каждая кнопка автоматически получает `ButtonView`: событие `Clicked`, троттлинг 0.15 c, press-анимацию и звук клика. По **имени** генератор предлагает декларативную навигацию (правится в окне):

| Имя кнопки | Предложение навигации |
|---|---|
| `button_close` | закрыть свой попап (вне попапа — `Pop`) |
| `button_<x>` при существующем `popup_<x>` на той же странице | открыть этот попап (`button_setting` → `popup_setting`) |
| `button_play` | `Show("gameplay")` |
| `button_<pageId>` | `Show(<pageId>)` (`button_main_menu` → `mainmenu`) |
| `button_restart`, `button_buy`, ... (без конвенции) | без предложения; в попапах работают через результат `OpenAsync` |

**Без правила:** безымянная кнопка не получит константу `UiIds` и поле вьюшки; кнопка с «неговорящим» именем потребует ручной подписки (`UiKit.OnClick` / `Clicked` в partial) — навигация сама не развесится.

## Счётчики (деньги, счёт, уровень)

| Правило | Пример |
|---|---|
| Панель: класс `fui_type_panel`, name `panel_<id>` | `panel_coin`, `panel_merges`, `panel_reward` |
| Внутри — Label с **числом в gradient-обёртке** | `<color=white><gradient="...">512</gradient></color>` |

- id счётчика = имя панели без `panel_` (`panel_coin` → `coin`); редактируется в окне.
- id, содержащий `score` → `ScoreView`, `level` → `LevelView`, остальное → `CounterView`.
- Панель с именем `*timer*` → `TimerView` (mm:ss / h:mm:ss, обратный отсчёт).
- Панель, внутри которой есть кнопка с числовым gradient-Label (цена) → `ShopItemView` (кнопка + цена + иконка + состояния куплено/недоступно); числа внутри кнопок **не** считаются счётчиками.
- Обновление значения подменяет только число внутри rich-text-обёртки — градиент и текст вокруг («LEVEL 5» → «LEVEL ») сохраняются.
- Один и тот же id на нескольких страницах — это **один** счётчик: `UiKit.Money.Set` обновит все вхождения. Семантически одинаковые панели с разными именами (`panel_merges` на экране и `panel_merges_count` в попапе) объединяются правкой id в окне.

**Без правила:** число не в gradient-обёртке или панель без `panel_`-имени → элемент останется обычным `LabelView`/`VisualElement`; счётчик можно назначить вручную override'ом в окне, но авто-детект — это именно эвристика по gradient-числам. Помните: эвристика может и ложно сработать — спорные случаи исключайте в окне.

## Прогресс-бары

| Правило | Пример |
|---|---|
| Класс `fui_type_progressbar`; fill — вложенный элемент с `fill` в имени (иначе берётся последний ребёнок) | бар загрузки на `loading` |

`BarView` задаёт ширину fill **в пикселях** от фактической ширины трека (в импортированном USS fill уже дизайн-ширины, процент от родителя дал бы неверную шкалу). `UiFakeLoading` находит первый `fui_type_progressbar` стартовой страницы сам.

**Без правила:** бар не распознается — двигать придётся руками через стили.

## Переключатели (звук, музыка, вибрация)

| Правило | Пример |
|---|---|
| Кнопка с name `toggle_<x>` | `toggle_sound`, `toggle_music` |

`toggle_*`-кнопка становится `ToggleView`: состояние on/off классом `is-off`, событие `Changed`, `SetWithoutNotify`. Рекомендация для попапа настроек — `toggle_sound` / `toggle_music` (или `button_sound` / `button_music`, если тумблер — обычная кнопка): подключение к аудио занимает две строки:

```csharp
SomeToggle.Changed += on => UiKit.Audio.SoundOn = on;
SomeToggle.SetWithoutNotify(UiKit.Audio.SoundOn);
```

**Без правила:** тумблер, собранный из голых image/label (как в исходном `MY_GAME_DESIGN`), не станет `ToggleView` — состояние придётся переключать вручную через типизированные поля (`SetTint`/`SetImage` + `UiKit.OnClick`).

## Прочие элементы

- `fui_type_label` → `LabelView` (включая Label внутри кнопок — цены доступны как поля с `SetText`, сохраняющим градиент).
- `fui_type_image` → `ImageView` (`SetImage(Sprite/Texture2D)`, `SetTint`, `Show/Hide`).
- Служебные `spacer_*` / `layout_group_*` по умолчанию не генерируются (настройка окна «включать служебные»); `layout_group_0001` (верх экрана) стоит указать как top-бар страницы для intro-анимации.
- Панели-обёртки `panel_panel_*` не участвуют в классификации виджетов.
- **Уникальность имён обязательна только в пределах скоупа** (экран / попап): `button_restart` может быть и в `popup_pause`, и в `popup_endgame` — пути разведут их (`gameplay/popup_pause/button_restart`). Дубль в одном скоупе → генератор построит вложенный путь через именованного предка, а C#-имя получит суффикс `_2`; неоднозначный путь в runtime даёт ошибку со списком кандидатов.
- Имена попадают в C# после санитизации: недопустимые символы → `_`, ведущая цифра → префикс `_`, ключевые слова экранируются; таблица «имя Figma → имя C#» видна в отчёте окна.

## Что деградирует без конвенций — сводка

| Нарушение | Последствие |
|---|---|
| Нет `fui_type_screen` | страницы нет вообще |
| Нет `fui_type_popup` | попап не скрывается, не открывается, перехватывает ввод |
| Безымянные элементы | нет полей вьюшки и констант `UiIds` |
| «Неговорящие» имена кнопок | навигацию развешивать вручную |
| Числа без gradient-обёртки | счётчики не найдены — override в окне или ручной `LabelView` |
| Нет `toggle_*` | переключатели настроек собираются вручную |
| Дубли имён в одном скоупе | суффиксы `_2` и длинные пути — читаемость страдает |

Худший случай (только экраны и кнопки) — kit всё равно даёт роутер, полное покрытие элементов типизированными полями и `UiKit.OnClick`; теряется только «магия» виджетов и автонавигации.
